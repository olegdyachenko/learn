Мы будем программировать на языке PHP. Его называют препроцессорным языком. PHP может собрать страницы из кусочков, как конструктор. При этом PHP превращает код собранных страниц в HTML. Это и есть препроцессинг.


статья для понимания - https://htmlacademy.ru/blog/education/what/php-what-for
в учебник по PHP - https://htmlacademy.ru/tutorial/php/basics



PHP удобен для разработки сайтов, потому что сам по себе HTML не умеет собирать страницы и автоматически менять на них информацию, а PHP умеет.


require('components/company_info.php'); - добавляем кусок кода на страницу


// - комментарии и /**/

<?php // Открывающий PHP-тег

require('file.php'); // Какой-то PHP-код

?>    // Закрывающий PHP-тег


альтернатива для require, include - https://www.php.net/manual/ru/function.include.php

Компонентом называется файл с кодом какого-то смыслового раздела страницы. Например, у нас есть компонент шапки, подвала, контента страницы.



 этом нам поможет склеивание файлов. Когда по ходу выполнения сценария встречается команда require, она заменяется на содержимое подключаемого файла, ровно в том порядке, в каком расположены require. Это и есть склеивание файлов.


 $product; // Объявили переменную $product
Имя переменной должно начинаться со знака доллара $, дальше может идти буква или символ подчёркивания. Название должно быть понятным для людей и описывать то, что находится в переменной.

$product = 'люстра'; // Присвоили переменной $product значение 'люстра'


<?php echo($product); ?>





Можно добавить переменную в шаблон знакомым способом, но такая запись слишком многословная:

<img src="<?php echo($img_url); ?>">

// Добавили переменную в атрибут картинки в шаблоне
В PHP есть запись покороче:

<img src="<?= $img_url ?>">
<-- Двойные кавычки в атрибуте не удаляются -->
